# Detections List V1

Обновлено: 2026-04-30
Статус: черновая экранная спецификация

## Назначение

Это первая детализация экрана списка обнаруженных сигналов.
Экран открывается из главного `Simple` экрана по нажатию `OK`.

Главные задачи:
- быстро показать, что именно найдено;
- помочь понять, какие сигналы важнее прямо сейчас;
- не перегрузить пользователя большим количеством деталей;
- дать быстрый вход в карточку выбранного сигнала.

## Роль Экрана В Сценарии

Этот экран является первым уровнем после события обнаружения.

Типичный путь:
1. Пользователь видит уведомление на главном экране.
2. Нажимает `OK`.
3. Попадает в список обнаружений.
4. Выбирает нужный сигнал.
5. Переходит в карточку сигнала или контекстное меню.

Экран должен одинаково хорошо работать:
- когда обнаружен один сигнал;
- когда сигналов несколько;
- когда сигнал только что появился;
- когда часть сигналов уже затухает.

## Главный Принцип

Это не таблица для инженера и не журнал логов.
Это экран приоритизации внимания.

Он должен помогать ответить на вопросы:
- что найдено;
- что из этого новое;
- что сильнее;
- что растет;
- что стоит открыть первым.

## Экран 320x480

Черновое деление по зонам:

```text
+--------------------------------+
| 1. STATUS BAR                28 |
+--------------------------------+
| 2. SCREEN TITLE              40 |
+--------------------------------+
| 3. FILTER / SUMMARY          36 |
+--------------------------------+
| 4. LIST AREA                292 |
|                                 |
|                                 |
|                                 |
|                                 |
|                                 |
+--------------------------------+
| 5. ACTION HINTS              84 |
|                                 |
+--------------------------------+
Total: 480 px
```

Рабочее распределение:
- статус-бар: `28 px`
- заголовок: `40 px`
- краткая сводка: `36 px`
- область списка: `292 px`
- нижние подсказки: `84 px`

## Структура Экрана

### 1. Status Bar

Сохраняем единый верхний статус:
- время;
- батарея;
- звук;
- вибрация;
- профиль `SIMPLE`;
- индикатор новых событий.

Пример:

```text
12:43   BAT 77%   VOL 2   VIB   SIMPLE   *
```

### 2. Screen Title

Здесь нужен короткий и спокойный заголовок:

```text
ОБНАРУЖЕННЫЕ СИГНАЛЫ
```

Если это полезно, справа можно держать число:

```text
ОБНАРУЖЕННЫЕ СИГНАЛЫ   4
```

### 3. Filter / Summary

В `Simple` режиме здесь лучше не делать сложные фильтры.
Достаточно короткой сводки:
- сколько сигналов всего;
- сколько новых;
- что сейчас в приоритете.

Пример:

```text
4 сигнала, новых 1, активных 3
```

или

```text
Приоритет: новый сильный сигнал
```

## Формат Одной Строки Списка

Одна запись должна быть компактной, но информативной.

Минимальный состав:
- признак выбора;
- тип сигнала;
- мощность;
- динамика;
- краткая строка-пояснение.

Черновой формат:

```text
> Analog video      -52 dBm   ↑
  Новый, уровень растет
```

Еще пример:

```text
  Beacon            -78 dBm   ↓
  Сигнал затухает
```

Если нужна еще более короткая запись:

```text
> Analog video   -52 dBm   ↑
  Новый
```

## Обозначения В Списке

Для быстрого чтения удобно использовать постоянные метки:
- `Новый` — только что появился;
- `Активен` — сигнал есть и не теряет важность;
- `Снижается` — мощность падает;
- `Игнор` — уведомления отключены, если такие записи вообще решите показывать;
- `Видео?` — возможно есть смысл попробовать открыть видеорежим.

Для динамики:
- `↑` растет;
- `=` стабилен;
- `↓` падает.

## Приоритет Порядка В Списке

Для `Simple` режима разумно сортировать так:
1. новые сигналы;
2. более сильные сигналы;
3. сигналы с растущим уровнем;
4. остальные активные;
5. затухающие внизу.

Практическое правило:
- пользователь первым должен видеть то, что появилось недавно и выглядит важным.

## Базовый Вид Экрана

Случай: несколько сигналов, один из них новый.

```text
+--------------------------------+
| 12:43 BAT77 VOL2 VIB SIMPLE  * |
+--------------------------------+
| ОБНАРУЖЕННЫЕ СИГНАЛЫ        4  |
+--------------------------------+
| 4 сигнала, новых 1, активных 3 |
+--------------------------------+
| > Analog video     -52 dBm   ↑ |
|   Новый, уровень растет        |
|                                |
|   Digital burst    -67 dBm   = |
|   Короткая активность          |
|                                |
|   Beacon           -78 dBm   ↓ |
|   Сигнал затухает              |
|                                |
|   Unknown source   -81 dBm   = |
|   Слабый стабильный сигнал     |
+--------------------------------+
| OK: открыть                    |
| Назад: сканер   Fn: игнор/звук |
+--------------------------------+
```

## Состояние: Один Сигнал

Если найден только один сигнал, экран все равно должен оставаться списком, но выглядеть спокойнее:

```text
+--------------------------------+
| 12:43 BAT77 VOL2 VIB SIMPLE  * |
+--------------------------------+
| ОБНАРУЖЕННЫЕ СИГНАЛЫ        1  |
+--------------------------------+
| 1 сигнал, новый 1              |
+--------------------------------+
| > Analog video     -52 dBm   ↑ |
|   Новый сигнал                |
|   Возможно видео              |
|                                |
|                                |
|                                |
|                                |
|                                |
|                                |
+--------------------------------+
| OK: открыть                    |
| Назад: сканер   Fn: mute       |
+--------------------------------+
```

## Состояние: Нет Активных Сигналов

Если пользователь открыл список, а сигналы уже исчезли, это не должно выглядеть как ошибка.

```text
+--------------------------------+
| 12:49 BAT76 VOL2 VIB SIMPLE   |
+--------------------------------+
| ОБНАРУЖЕННЫЕ СИГНАЛЫ        0  |
+--------------------------------+
| Сейчас активных сигналов нет   |
+--------------------------------+
| Список пуст                    |
| Сигналы исчезли или затухли    |
|                                |
| Вернуться к сканированию       |
|                                |
|                                |
|                                |
|                                |
+--------------------------------+
| OK: назад                      |
| Назад: сканер                  |
+--------------------------------+
```

## Состояние: Слишком Много Сигналов

Если записей больше, чем помещается на экран:
- список должен скроллиться энкодером;
- сверху и снизу полезно показывать, что есть еще элементы;
- на экране лучше держать 3-4 полноценных карточки, а не 8 тесных строк.

Признак продолжения списка можно показывать так:

```text
... выше
... ниже
```

или через графические маркеры.

## Поведение Выбора

Когда пользователь крутит энкодер:
- перемещается выделение;
- выбранная строка получает четкий акцент;
- при необходимости список плавно прокручивается.

Выделенная запись должна отличаться очень ясно:
- стрелкой `>`;
- инверсией;
- рамкой;
- более ярким фоном.

Для `Simple` режима я бы предпочел:
- стрелка `>`;
- слегка выделенная подложка;
- без слишком резких эффектов.

## Нажатия И Переходы

Основная логика:
- `OK` -> открыть выбранный сигнал;
- `Назад` -> вернуться на главный экран;
- `Fn` -> быстрое вторичное действие.

Варианты для `Fn`:
- `mute` текущего уведомления;
- временно скрыть карточку нового события;
- быстро игнорировать выбранный сигнал.

Самый безопасный вариант для первой версии:
- `Fn` только глушит звук;
- игнорирование оставить внутри карточки сигнала, чтобы не делать случайных действий в списке.

## Какие Данные Показывать В Одной Записи

Для первой версии достаточно:
- имя/тип сигнала;
- мощность;
- динамика;
- краткий статус.

Что лучше не показывать здесь:
- точную частоту длинной строкой;
- историю изменения уровня;
- длинные пояснения;
- несколько служебных параметров сразу.

Частоту лучше перенести в карточку сигнала.

## Правила Текста

Текст должен быть очень коротким.

Подходящие формулировки:
- `Новый сигнал`
- `Уровень растет`
- `Сигнал затухает`
- `Слабый стабильный`
- `Возможно видео`

Неподходящие формулировки:
- длинные технические описания;
- сложные сокращения без пользы;
- несколько мыслей в одной строке.

## Минимальный Набор Данных Для Прошивки

Чтобы экран заработал, нужны:
- общее число активных сигналов;
- число новых сигналов;
- индекс выбранного сигнала;
- массив отображаемых записей, где каждая запись содержит:
  - `type_name`;
  - `power_dbm`;
  - `trend`;
  - `is_new`;
  - `is_active`;
  - `short_hint`.

Опционально:
- `can_show_video`;
- `priority_score`;
- `is_ignored`.

## Связь С Следующим Экраном

Этот экран должен вести в карточку сигнала.
Карточка уже может показывать:
- тип;
- частоту;
- мощность;
- динамику;
- статус;
- доступные действия.

## Вывод Для Следующего Шага

Теперь логично детализировать:
- `карточку сигнала`;
- или сразу `карточку сигнала + контекстное меню`.

Самый полезный следующий экран: карточка сигнала с действиями, потому что именно туда ведет `OK` из списка.
