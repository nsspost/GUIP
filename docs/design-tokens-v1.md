# Design Tokens V1

Обновлено: 2026-04-30
Статус: рабочая база для реализации

## Назначение

Этот документ переводит визуальные концепты в инженерный набор правил для реализации интерфейса на `LVGL`.

Цель:
- зафиксировать единый визуальный язык;
- избежать случайного расползания стилей по экранам;
- дать понятную основу для первой реализации на дисплее `320x480`;
- учитывать реальные ограничения `RGB565`, `STM32` и небольшого вертикального экрана.

## Основной Визуальный Принцип

Интерфейс должен быть:
- темным;
- спокойным в обычном режиме;
- выразительным при тревоге;
- читаемым на расстоянии и с первого взгляда;
- достаточно "живым", но без тяжелых эффектов.

Ключевая идея:
- в фоне интерфейс нейтрален;
- данные живут за счет иерархии;
- события выделяются цветом, контрастом и формой, а не обилием декора.

## Базовая Цветовая Система

Ниже даны ориентиры в `HEX`.
Для реальной реализации их стоит сразу перевести в `RGB565`.

### Базовые Фоны

- `bg_root`: `#111418`
- `bg_surface`: `#171B20`
- `bg_panel`: `#1D232A`
- `bg_panel_soft`: `#222A32`
- `bg_selected_soft`: `#2A3138`

Назначение:
- `bg_root` для общего фона экрана;
- `bg_surface` для крупных зон;
- `bg_panel` для карточек и панелей;
- `bg_panel_soft` для вторичных контейнеров;
- `bg_selected_soft` для мягкого выделения без тревоги.

### Текст

- `text_primary`: `#E7EDF3`
- `text_secondary`: `#AEB8C2`
- `text_muted`: `#7E8A96`
- `text_inverse`: `#0F1216`

Назначение:
- `text_primary` для главных заголовков и ключевых значений;
- `text_secondary` для подзаголовков и пояснений;
- `text_muted` для подсказок и вторичной информации;
- `text_inverse` для текста на ярких акцентных плашках.

### Акценты

- `accent_cyan`: `#4FD1D9`
- `accent_cyan_soft`: `#2B8E96`
- `accent_amber`: `#E5A93B`
- `accent_amber_soft`: `#8E6926`
- `accent_green`: `#63C174`
- `accent_red`: `#D85C5C`

Назначение:
- `accent_cyan` для живой активности, спектра, графиков, активного анализа;
- `accent_amber` для новых событий, выбора, тревожных акцентов;
- `accent_green` для нормального состояния и подтверждения;
- `accent_red` только для ошибок или тяжелых неисправностей.

## Практические Ограничения Для RGB565

Так как экран работает в `RGB565`, нужно держать в голове:
- очень тонкие переходы и мягкие тени будут выглядеть грубее, чем в мокапе;
- слишком близкие оттенки серого могут сливаться;
- мелкие полупрозрачные детали почти всегда проигрывают крупным формам;
- цветовую палитру лучше держать компактной.

Практическое правило:
- использовать не более 2-3 уровней фонового серого;
- не делать еле заметных границ;
- проверять контраст не на белом макете, а на реальном темном фоне.

## Типографика

Для первой версии стоит исходить из ограниченного набора размеров.

### Размеры

- `font_xl`: `28 px`
- `font_l`: `22 px`
- `font_m`: `18 px`
- `font_s`: `16 px`
- `font_xs`: `14 px`

### Применение

- `font_xl`
  - главный статус на ключевых экранах;
  - очень важные числа или заголовки тревоги.

- `font_l`
  - заголовки экранов;
  - основные метрики.

- `font_m`
  - обычные значения;
  - названия сигналов;
  - кнопочные подсказки.

- `font_s`
  - вторичные пояснения;
  - подписи строк списка;
  - строки статуса.

- `font_xs`
  - только для очень вторичной информации;
  - использовать редко.

### Правила

- не опускаться ниже `14 px`;
- длинные строки лучше сокращать, а не уменьшать шрифт;
- на `Simple` экранах предпочитать `18 px` и выше;
- если сомнение между красивее и крупнее, выбирать крупнее.

## Вес И Стиль Шрифта

Если будет доступен только один шрифт, этого достаточно.
Нужны 2 веса:
- `regular`
- `semi-bold`

Применение:
- `semi-bold` для заголовков, чисел, активных состояний;
- `regular` для пояснений и вторичного текста.

## Базовая Сетка И Отступы

Для `320x480` удобна простая сетка с шагом `4 px`.

### Шаги

- `space_1`: `4 px`
- `space_2`: `8 px`
- `space_3`: `12 px`
- `space_4`: `16 px`
- `space_5`: `20 px`
- `space_6`: `24 px`

### Практика

- внутренние отступы карточек: `12-16 px`
- расстояние между крупными блоками: `12-16 px`
- расстояние между строками внутри блока: `8 px`
- отступы от краев экрана: `12 px`

## Радиусы И Формы

Чтобы интерфейс оставался приятным, но не выглядел "телефонным", радиусы должны быть умеренными.

### Радиусы

- `radius_s`: `6 px`
- `radius_m`: `10 px`
- `radius_l`: `14 px`

### Применение

- маленькие плашки и метки: `6 px`
- карточки, панели, строки списка: `10 px`
- крупные мягкие контейнеры: `14 px`

Правило:
- не делать идеально круглых и "пузатых" элементов;
- форма должна быть мягкой, но собранной.

## Толщины Линий И Границ

- `border_subtle`: `1 px`
- `border_strong`: `2 px`

Применение:
- `1 px` для большинства разделений;
- `2 px` для активного выбора или тревожного состояния.

Цвета границ:
- `line_soft`: `#2D3740`
- `line_focus`: `#E5A93B`
- `line_info`: `#4FD1D9`

## Высоты Типовых Элементов

### Status Bar

- высота: `28 px`
- текст: `14-16 px`
- горизонтальный padding: `8-12 px`

### Header Block

- высота: `40-56 px`
- главный текст: `22 px`
- вторичный: `16 px`

### Card / Panel

- внутренний padding: `12-16 px`
- межстрочный интервал: `8 px`

### List Item

Для списка обнаружений:
- высота одной компактной записи: `56-64 px`
- высота записи с более свободной подачей: `68-76 px`

Для первой версии:
- лучше целиться в `64-72 px` на запись;
- это даст 3-4 записи на экран без потери читаемости.

### Bottom Hints Area

- высота: `72-84 px`
- текст: `16-18 px`

## Состояния Элементов

### Спокойное Состояние

- фон нейтральный;
- текст нормальной контрастности;
- без янтарных акцентов;
- активность выражается цианом.

### Новое Событие

- допускается янтарная плашка, рамка или маркер;
- но не нужно делать весь экран тревожным;
- выделяется только самое важное место.

### Выбранный Элемент

- мягкая подложка;
- янтарный маркер;
- более яркий текст;
- при желании тонкая рамка `2 px`.

### Ошибка

- красный только точечно;
- не использовать красный как обычный акцент;
- ошибки должны быть редкими и заметными.

## Иконки И Маркеры

Для первой версии иконки лучше использовать очень экономно.

Что действительно полезно:
- батарея;
- звук / mute;
- вибрация;
- стрелка динамики `↑ = ↓`;
- маленький маркер нового события.

Правило:
- если текст понятнее и дешевле по реализации, текст лучше иконки.

## Графики И Спектр

### Simple Режим

На главном экране:
- не полноценный спектроанализатор;
- а "живая полоса активности".

Правила:
- 1 графическая зона;
- 1 акцентный цвет;
- широкие формы;
- без мелкой сетки;
- без длинной шкалы частот.

### Pro Режим

На спектральном экране:
- можно добавить сетку;
- можно дать подписи частот;
- можно дать zoom и панораму;
- но подписи должны быть редкими и крупными.

## Токены Для Списка Обнаружений

### Список

- `list_gap`: `8 px`
- `list_item_radius`: `10 px`
- `list_item_padding`: `12 px`
- `list_item_selected_border`: `2 px`

### Текст Списка

- название сигнала: `18 px semi-bold`
- мощность: `18 px semi-bold`
- подпись статуса: `16 px regular`
- summary line сверху: `16 px regular`

### Цветовая Логика

- новый сигнал: янтарный акцент;
- активный сигнал: обычный светлый текст;
- затухающий: светлый текст + muted secondary line;
- игнорируемый: только в специальных местах, не на главном списке первой версии.

## Токены Для Карточки Сигнала

- title: `22 px semi-bold`
- metrics: `18 px`
- secondary note: `16 px`
- action row height: `44-48 px`
- action row radius: `8-10 px`

## Токены Для Bottom Hints

Нижние подсказки физическим кнопкам должны быть:
- всегда на месте;
- короткими;
- не длиннее 2-3 слов на действие.

Примеры:
- `OK открыть`
- `Назад меню`
- `Fn звук`

Размер:
- `16 px` или `18 px`

## Рекомендации Для LVGL

### Что Стоит Делать

- использовать повторяемые стили `lv_style_t` для панелей, списков, статусов и подсказок;
- вынести палитру и размеры в отдельный header с токенами;
- сразу делать общие компоненты:
  - status bar
  - screen header
  - panel card
  - detection list item
  - bottom hints

### Чего Лучше Избегать В Первой Версии

- тяжелых теней;
- сложных прозрачностей;
- большого числа уникальных цветов;
- слишком тонких границ;
- большого количества мелких иконок.

### Практический Путь

1. Сначала собрать темing и базовые стили.
2. Потом реализовать status bar и bottom hints как общие блоки.
3. После этого собрать `Simple Home`.
4. Затем `Detections List`.
5. Потом `Signal Details` и `Hold Monitor`.
