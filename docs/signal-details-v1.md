# Signal Details V1

Обновлено: 2026-04-30
Статус: черновая экранная спецификация

## Назначение

Экран карточки сигнала открывается из списка обнаружений.
Он нужен, чтобы показать более полную информацию по выбранному сигналу и дать безопасный доступ к действиям.

## Основные Задачи

- показать основные свойства сигнала;
- явно обозначить его текущее состояние;
- дать короткий набор действий;
- не заставлять пользователя принимать необратимое действие случайно.

## Состав Данных

Что стоит показывать:
- тип сигнала;
- частота или диапазон;
- мощность;
- динамика;
- статус;
- краткая пометка, почему сигнал интересен;
- доступность видео, если это уместно.

## Экран 320x480

```text
+--------------------------------+
| STATUS BAR                   28 |
+--------------------------------+
| TITLE + SIGNAL NAME          44 |
+--------------------------------+
| PRIMARY METRICS             120 |
+--------------------------------+
| STATUS / SHORT NOTE          64 |
+--------------------------------+
| ACTION MENU                 140 |
+--------------------------------+
| HINTS                        84 |
+--------------------------------+
```

## Базовый Вид

```text
+--------------------------------+
| 12:44 BAT77 VOL2 VIB SIMPLE   |
+--------------------------------+
| SIGNAL: Analog video          |
+--------------------------------+
| Частота: 1.2 GHz              |
| Уровень: -52 dBm              |
| Динамика: растет              |
| Статус: активен               |
+--------------------------------+
| Новый интересный сигнал       |
| Возможно есть видеоданные     |
+--------------------------------+
| > Остаться на частоте         |
|   Показать видео / USB        |
|   Игнорировать сигнал         |
|   Назад                       |
+--------------------------------+
| OK: выбрать                   |
| Назад: список   Fn: mute      |
+--------------------------------+
```

## Логика Действий

Для первой версии набор действий видится таким:
- `Остаться на частоте`
- `Показать видео / USB`
- `Игнорировать сигнал`
- `Назад`

Если действие рискованное или неочевидное:
- нужна короткая промежуточная проверка;
- особенно для `Игнорировать сигнал`.

## Подтверждение Игнорирования

Лучше не выполнять игнор мгновенно.
Подходящий диалог:

```text
Игнорировать этот сигнал?

> Да
  Нет
```

## Минимальные Данные Для Прошивки

- `type_name`
- `frequency_text`
- `power_dbm`
- `trend`
- `status_text`
- `short_reason`
- `can_show_video`
- `can_hold_frequency`

## Следующий Экран

Из этой карточки пользователь чаще всего пойдет:
- в `hold monitor`;
- в `video view`;
- обратно в список.
