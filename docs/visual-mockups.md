# Visual Mockups

Обновлено: 2026-04-30
Статус: визуальная база для реализации

## Назначение

Здесь собраны статические мокапы экранов, которые можно использовать как визуальную основу при реализации интерфейса на `LVGL`.

Важно:
- это не финальные pixel-perfect спецификации;
- это визуальные ориентиры по композиции, стилю, акцентам и иерархии;
- для реальной реализации их нужно адаптировать под `320x480`, `RGB565` и ограничения производительности.

## Набор Мокапов

### 1. Ранний концепт: главный экран + список обнаружений

Файл:
- `assets/mockups/concept-simple-home-and-detections.png`

Назначение:
- зафиксировать общее ощущение премиального и органичного интерфейса;
- использовать как reference по визуальному языку.

### 2. Более реалистичный вариант: главный экран + список обнаружений

Файл:
- `assets/mockups/realistic-simple-home-and-detections.png`

Назначение:
- показать более честную адаптацию под реальный `320x480`;
- использовать как reference по плотности информации и размеру элементов.

### 3. Гибридный вариант: главный экран + список обнаружений

Файл:
- `assets/mockups/hybrid-simple-home-and-detections.png`

Назначение:
- совместить приятность раннего концепта с реалистичностью под embedded-дисплей;
- использовать как основной кандидат на стиль первой версии UI.

### 4. Гибридный вариант: карточка сигнала + удержание на частоте

Файл:
- `assets/mockups/hybrid-signal-details-and-hold-monitor.png`

Назначение:
- продолжить основной пользовательский сценарий;
- использовать как базу для экранов `Signal Details` и `Hold Monitor`.

## Рекомендуемое Практическое Использование

Если брать эти материалы в работу, то логично использовать их так:
- `hybrid-simple-home-and-detections.png` как главный stylistic reference;
- `realistic-simple-home-and-detections.png` как ограничитель по плотности и читаемости;
- `hybrid-signal-details-and-hold-monitor.png` как ориентир для второго шага сценария после обнаружения сигнала.

## Следующие Полезные Шаги

- на основе `hybrid`-мокапов подготовить более точные LVGL-friendly layout specs;
- собрать базовую дизайн-систему: цвета, размеры, радиусы, типографика, отступы;
- перейти к первому каркасу экранов в коде.
