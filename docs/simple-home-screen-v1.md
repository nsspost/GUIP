# Simple Home Screen V1

Обновлено: 2026-04-30
Статус: черновая экранная спецификация

## Назначение

Это первая детализация главного экрана для `Simple` режима.
Задача этого экрана:
- показать, что прибор включен и жив;
- показать, что сканирование идет;
- не перегрузить пользователя спектральной аналитикой;
- быстро сообщить о новом обнаружении;
- дать быстрый вход в список обнаружений и меню.

## Роль Экрана В Продукте

Этот экран должен быть экраном "по умолчанию".
Пользователь большую часть времени находится именно здесь.

Экран должен хорошо работать в трех состояниях:
- спокойное наблюдение;
- появилось новое обнаружение;
- есть активные обнаружения, но без новой тревоги прямо сейчас.

## Экран 320x480

Ниже грубое деление по зонам для портретного экрана `320x480`.

```text
+--------------------------------+
| 1. STATUS BAR                28 |
+--------------------------------+
| 2. DEVICE STATE              52 |
+--------------------------------+
| 3. LIVE ACTIVITY AREA       220 |
|                                 |
|                                 |
|                                 |
|                                 |
+--------------------------------+
| 4. DETECTIONS SUMMARY        96 |
|                                 |
|                                 |
+--------------------------------+
| 5. SOFT HINTS / BUTTONS      84 |
|                                 |
+--------------------------------+
Total: 480 px
```

Рабочее распределение высот:
- статус-бар: `28 px`
- зона состояния прибора: `52 px`
- живая область: `220 px`
- зона обнаружений: `96 px`
- нижние подсказки/кнопки: `84 px`

Это не финальная сетка, а стартовая.

## Состав Зон

### 1. Status Bar

Что показываем постоянно:
- время;
- батарея;
- звук;
- вибрация;
- профиль `SIMPLE`;
- индикатор новых событий.

Пример строки:

```text
12:41   BAT 78%   VOL 2   VIB   SIMPLE   *
```

Логика:
- если новых событий нет, индикатор `*` скрыт;
- если звук отключен, вместо `VOL 2` показывать `MUTE`;
- если вибрация отключена, вместо `VIB` показывать `VIB OFF`.

### 2. Device State

Эта зона отвечает на вопрос "что делает прибор прямо сейчас".

Что здесь должно быть:
- крупный статус;
- текущий режим работы;
- краткая строка пояснения.

Пример:

```text
СКАНИРОВАНИЕ
Широкий диапазон
Эфир анализируется
```

Другие возможные статусы:
- `УДЕРЖАНИЕ ЧАСТОТЫ`
- `НОВОЕ ОБНАРУЖЕНИЕ`
- `ПРОСМОТР ВИДЕО`
- `ПАУЗА`
- `ОШИБКА`

### 3. Live Activity Area

Это главная "живая" зона экрана.
В `Simple` режиме здесь не нужен точный аналитический спектр.
Нужна легкая и наглядная живая индикация работы.

Что можно показывать:
- упрощенный бегущий спектр;
- мягкую анимацию уровней;
- несколько пиков, меняющихся со временем;
- индикатор sweep или активности диапазона.

Главная цель:
- пользователь должен видеть, что устройство не зависло;
- экран должен создавать ощущение реального анализа эфира.

Чего здесь не делать:
- не выводить мелкие частотные шкалы;
- не перегружать числами;
- не делать сложную сетку.

### 4. Detections Summary

Это зона событий и краткого списка.

В ней должны жить:
- число активных обнаружений;
- последнее или самое важное обнаружение;
- краткий признак важности;
- предложение нажать `OK`, если есть смысл открыть список.

Когда ничего нет:

```text
Обнаружений нет
Последние 12 мин без событий
```

Когда есть активные сигналы:

```text
Активных обнаружений: 3
Новый: Analog video   -52 dBm   рост
OK: открыть список
```

### 5. Soft Hints / Buttons

Это нижняя зона подсказок по управлению.
Даже если в устройстве физические кнопки, экрану полезно объяснять, что сейчас делает каждая основная кнопка.

Базовый вариант:

```text
OK: обнаружения
Назад: меню
Fn: звук / режим
```

Если в текущем состоянии доступно другое действие, подсказки меняются контекстно.

## Вид При Новом Обнаружении

Состояние: только что найден сигнал, нужно быстро привлечь внимание.

```text
+--------------------------------+
| 12:43 BAT77 VOL2 VIB SIMPLE  * |
+--------------------------------+
| НОВОЕ ОБНАРУЖЕНИЕ              |
| Сигнал найден                  |
| Требует внимания               |
+--------------------------------+
|                                |
|      ~~~  ~~  ~~~~  ~~         |
|   ~~   ~~~~ ~~   ~~~~          |
|                                |
|   [ Analog video ]             |
|   Уровень: -52 dBm             |
|   Динамика: растет             |
|                                |
+--------------------------------+
| Активных обнаружений: 3        |
| OK: открыть список             |
| Fn: отключить звук             |
+--------------------------------+
| OK: подробнее                  |
| Назад: позже     Fn: mute      |
+--------------------------------+
```

Что важно в этом состоянии:
- статус и карточка нового события должны визуально доминировать над фоновой анимацией;
- экран не должен превращаться в длинное описание;
- одно новое событие показывается как приоритетное, даже если сигналов несколько.

## Поведение Элементов

### Status Bar

- обновляется всегда;
- батарея и время должны быть статичны по положению;
- индикатор новых событий должен появляться мгновенно.

### Live Activity Area

- анимация не должна быть слишком быстрой;
- в фоне она должна быть мягкой, чтобы не раздражать при длительном наблюдении;
- при тревоге можно кратко усиливать контраст или амплитуду.

### Detections Summary

- если событий нет, показывать спокойный текст;
- если есть новые события, показывать приоритетную карточку;
- если сигналов много, не пытаться уместить длинный список, а показывать только верхние 1-2 записи и счетчик общего количества.

## Навигация С Этого Экрана

Из `Simple` главного экрана должны быть доступны только самые нужные переходы:
- `OK` -> список обнаружений;
- `Назад` или `Menu` -> меню и настройки;
- `Fn` -> быстрое действие;
- длительное нажатие `Fn` -> переключение `Simple/Pro`, если вы решите именно так сделать.

## Что Не Показывать В Simple

Чтобы режим действительно оставался простым, здесь лучше не показывать:
- точную частотную сетку;
- детальные уровни по нескольким каналам сразу;
- длинные подписи типов сигналов;
- сложную историю событий;
- диагностические счетчики.

## Минимальный Набор Данных Для Реализации

Для первой версии этого экрана прошивке достаточно отдавать:
- текущее время;
- заряд батареи;
- признак звука и вибрации;
- текущий режим прибора;
- массив значений для простой анимации live area;
- число активных обнаружений;
- данные по самому важному или самому новому сигналу:
  - тип;
  - мощность;
  - динамика;
  - признак "новый".

## Вывод Для Следующего Шага

Если идти последовательно, после этого экрана логично делать один из двух вариантов:
- `список обнаруженных сигналов`;
- `карточку сигнала и контекстное меню`.

Самый естественный следующий экран: список обнаружений, потому что именно туда ведет `OK` с главного экрана.
